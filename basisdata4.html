Konsep Transaksi

Transaksi adalah suatu unit dari eksekusi program yang memungkinkan untuk mengakses dan melakukan perubahan pada beberapa data item. Suatu sistem basis data harus memastikan eksekuasi transaksi yang sesuai â€“ apakah keseluruhan transaksi dieksekusi atau tidak satupun operasi di dalamnya yang dieksekusi. Lebih jelasnya, basis data harus mengelola eksekusi transaksi yang terjadi secara berbarengan sehingga tidak terjadi ketidak konsistenan data pada basis data.

Transaction mengakses data menggunakan dua operasi, yaitu :

read (x)

Mentransfer data item X dari basis data ke local buffer yang merupakan bagian dari transaksi yang melakukan operasi read.

write (x)

Mentransfer data item X dari local buffer pada transaksi yang melakukan penulisan data kembali ke basis data.




ACID Property

Terdapat empat sifat penting dalam transaksi yang harus dijamin oleh DBMS dalam menjaga agar data tetap konsisten, yaitu :

  • Atomicity : apakah semua aksi dilaksanakan atau tidak sama sekali. User tidak perlu khawatir mengenai efek dari transaksi yang gagal.

  • Consistency : Setiap transaksi, dijalankan sendiri tanpa eksekusi transaksi lain pada saat yang bersamaan, harus menjaga consistency dari basis data.

  • Isolation : User harus dapat menjalankan suatu transaksi tanpa harus memperhatikan efek dari transaksi lain yang dijalankan bersamaan, bahkan jika DBMS melakukan interleave terhadap transaksi untuk alasan performansi.

  • Durability : Sekali DBMS memberitahukan user bahwa suatu transaksi telah dijalankan dengan sukses, pengaruhnya harus tetap bahkan jika terjadi crash pada sistem sebelum semua perubahan tersebut dilakukan pada disk.

    • Keempat properti tersebut sering di kenal sebagai ACID properties. Istilah ACID sering digunakan untuk menggambarkan keempat sifat transaksi tersebut : atomicity, consistency, isolation, dan durability. Dalam menjaga integritas data, sistem harus dapat memelihara keempat sifat tersebut dimana :


  • Isolation (+ Consistency) => Concurrency Control



  • Atomicity + Durability => Recovery





    • Contoh Kasus


    Jika Richard ingin mentransfer uang ke vanesha melalui via ATM . Richard akan mentransfer sebesar Rp. 100.000 dari bank A ke vanesha (bank B) . maka prosesnya akan seperti dibawah ini :

    kita misalkan bahwa : richard = saldo_Richard dan vanesha = saldo_Vanesha

    1. read(saldo_Richard)

    2. saldo_Richard := saldo_Richard - 100.000

    3. write(saldo_Richard)

    4. read(saldo_Vanesha)

    5. saldo_Vanesha := saldo_Vanesha + 100.000

    6. write(saldo_Vanesha)


    Penjelasan proses transaksi diatas :

    1. Disini system akan membaca saldo pada saldo_Richard

    2. Setelah system membaca saldo, maka system akan memotong / mengurangi saldo_Richard sesuai instruksi ( Rp. 100.000 )

    3. Setelah itu, system akan menampilkan hasil saldo_Richard yang sudah dikurangi pada nomor 2

    4. System akan membaca pada saldo_Vanesha

    5. Lalu system akan menambahkan saldo_Vanesha sesuai yang saldo_Richard kirimkan , yakni sebesar Rp. 100.000

    6. Setelah itu, maka system akan menampilkan hasil saldo_Vanesha yang sudah ditambahi sesuai pada nomor 5



    Berikut adalah penjelasan properti ACID yang terjadi pada proses transaksi di atas :

  • Consistency Requirement : Jumlah pada saldo_Richard dan saldo_Vanesha tidak berubah setelah eksekusi transaksi

  • Atomicity Requirement : Jika transaksi gagal dijalankan setelah Langkah ke 3 dan sebelum Langkah ke 6, maka system akan menjamin perubahan yang terjadi tidak direfleksikan di database, jika hal ini tidak terjadi maka akan inconsistency.

  • Isolation Requirement : Jika pada Langkah no 3 dan 6, transaksi lain T2 diiznkan untuk mengkases database yang diperbaharui sebagaian, maka database akan terlihat tidak konsisten. Isolasi dapat dilakukan hanya menjalankan transaksi secara satu demi satu ( serial )

  • Durability Requirement : Setelah saldo_Richard diberi tahu bahwa transaksi telah selesai ( sudah mengirim Rp. 100.000 kepada saldo_Vanesha), maka database harus tetap terupdate meskipun ada terjadi kegagalan pada saat terjadinya transaksi
